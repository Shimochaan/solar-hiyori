# ☀️ ソーラー発電日和 — STATUS

## アイディアのタイプ
**学習・趣味型**(将来的に検証型 → 収益型/プロダクト化を狙える)
- 一次目的: React の実装を学びながら、楽しく毎日見たくなるソーラー予報アプリを形にする
- 将来: PWA / React Native(Expo)で iOS・Android 化も視野

## ビジョン
> 屋根を遊ばせておくのは、油田を放置するのと同じだ。

「太陽光パネルは優秀なツールはあるが、"楽しくて毎日見ちゃう"ソーラー予報アプリはまだない」を狙う。

## 何ができるアプリか
1. **住所検索**: 今の家でも、これから買う土地でも、地名で場所を指定
2. **パネル想定**: 面積(㎡)・変換効率・性能比(PR)・電気単価をスライダーで設定
3. **今日の発電日和指数**: ☀️×5 の5段階スコア + 今日の予想発電量・金額
4. **脳内ソーラーパネル設置モード**: まだ設置していなくても「今月ここに置いていたら累計○円稼げていた」をバーチャル表示(土地探し中の人を惹きつけるフック)
5. **年間見積もり + 月別グラフ**: 過去1年の実測日射量から年間発電量と月別内訳
6. **今日の発電カーブ**: 時間別日射量の山なりグラフ
7. **計算の種明かし**: 使っている式を画面に明記(透明性)

## 計算ロジック(核心)
```
発電量(kWh) = 面積A(㎡) × 変換効率η × 日射量H(kWh/㎡) × 性能比PR
```
- 日射量 H は Open-Meteo の `shortwave_radiation`(MJ/m² を ÷3.6 で kWh/m² に変換)
- 検証済み: 千葉・20㎡・η20%・PR0.8 → 年間約4,797kWh(4kWpの実勢に一致)

## 技術スタック
- Vite + React 19 + Tailwind CSS v4 + Recharts
- データ: **Open-Meteo**(無料・APIキー不要・CORS対応 → 完全フロントエンドで完結)
  - Geocoding API(住所→緯度経度)
  - Forecast API(`past_days` で過去実測+予報)
  - Archive API(過去1年=年間見積もり)

## v2 構想(試作品レビュー 2026-06-12)
プロダクトの軸を「毎日見る予報」→「土地購入の投資判断ツール」へ進化させる。

1. **住所の粒度を番地単位に** — 土地購入は番地で選ぶ。市区町村粒度では意味が薄い。
   → 番地ジオコーディングが必要(無料の国土地理院API候補 / Google Mapsは有料)。
2. **入力住所を地図表示** — ピンで場所を見せて納得感を出す(Leaflet+OSMなら無料・キー不要)。
3. **投資回収シミュレーション(核心価値)** — パネル設置費用 vs 月別の稼ぎ →
   何年で投資回収できるか。蓄電池に貯める前提も含め、電気代がお得になる時期、
   家庭にとってどんな買い物かを示す。
4. **家族構成・暮らし方の入力** — #3の精度に必要(自家消費 vs 売電の比率が変わる)。
   何人家族・在宅パターンなどから電力消費プロファイルを推定。
5. **(将来)不動産資料PDF/画像を入れるだけで計測** — 近隣建物の遮蔽も加味。
   → OCR/画像認識+太陽軌道+遮蔽モデルが要る最先端R&D。長期目標。

### 重要な技術的事実(設計に効く)
- **Open-Meteoの日射量は数km格子の値**。番地単位で緯度経度を取っても、隣の土地と
  日射量の数値自体は同じになる。番地精度が本当に効くのは「地図表示の納得感」と
  「#5の近隣建物の遮蔽計算」。日射量そのものの精度向上ではない点に注意。
